# Plan: Add Water Shader to Landing Page

## Context

The site has a fixed background SVG at `z-index: -1` (in `GlobalLayout`) and page content in normal flow on top. The goal is to add a `@paper-design/shaders-react` Water shader that sits **between** the SVG backdrop and the page content — visible through the transparent `homeContainer` but not blocking interaction. It must lazy-load after initial paint to avoid hurting Lighthouse scores.

## Layering

```
z-index: -1  │  globalBackground div (fixed SVG backdrop)     ← GlobalLayout.css.ts:32
z-index:  0  │  Water shader (fixed, pointer-events: none)    ← NEW
z-index:  1+ │  Page content (normal flow)                    ← existing
```

The `homeContainer` already has `background: transparent`, so the shader will show through without changes to content styles.

## Steps

### 1. Install the package

```bash
npm install @paper-design/shaders-react --workspace=@santan/frontend
```

### 2. Create `apps/frontend/src/components/WaterShader/WaterShader.css.ts`

Vanilla Extract style for the shader container:
- `position: fixed`, `top: 0`, `left: 0`, `width: 100vw`, `height: 100vh`
- `zIndex: 0` — above the SVG backdrop (`-1`) but below normal-flow content
- `pointerEvents: 'none'` — non-interactive overlay
- Opacity transition: start at `0`, animate to `1` for a smooth fade-in after load

### 3. Create `apps/frontend/src/components/WaterShader/WaterShader.tsx`

Default-export a component that:
- Renders `<Water>` from `@paper-design/shaders-react`
- Passes `style={{ width: '100%', height: '100%' }}` to fill its container
- Uses the shader container style from step 2
- Uses a `useState` + `useEffect` to flip opacity from 0 to 1 after mount (fade-in)
- Props from the issue sample: `speed`, `colorBack`, `colorHighlight`, `highlights`, `layering`, `edges`, `waves`, `caustic`, `size` — no `image` prop initially
- Marks the container `aria-hidden="true"`

### 4. Modify `apps/frontend/src/pages/Home/Home.tsx`

In the `HomePage` component (not the inner `Home` — it needs to render regardless of preview mode):

- Add `React.lazy(() => import("@/components/WaterShader/WaterShader.tsx"))` — follows the existing `PreviewWrapper` lazy pattern on line 17
- Add a `useEffect` + `useState` gate: set `showShader = true` after mount (defers the import until after first paint)
- Render inside `<Suspense fallback={null}>` **before** the existing return JSX, so it layers beneath content:

```tsx
{showShader && (
  <Suspense fallback={null}>
    <WaterShader />
  </Suspense>
)}
```

Place this just before the ternary that renders either `PreviewWrapper` or `HomePublished`.
